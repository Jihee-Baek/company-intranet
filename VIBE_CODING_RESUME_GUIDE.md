# 바이브 코딩(Vibe Coding) 경력기술서 작성 가이드

'바이브 코딩'은 AI와 협업하여 자연어로 의도를 전달하고 제품을 만드는 고생산성 개발 방식입니다. 이를 경력기술서에 효과적으로 녹여내기 위한 가이드를 제공합니다.

---

## 1. 핵심 역량 (Summary)
**전략:** 자신을 'AI 도구를 완벽히 다루는 고효율 개발자'로 정의합니다.

*   **키워드:** AI-Native 개발 방식, AI 에이전트 오케스트레이션, 개발 생산성 극대화, 신속한 프로토타이핑.
*   **작성 예시:** 
    > "LLM 기반 AI 에이전트(Cursor, Gemini CLI 등)를 활용한 **'Vibe Coding' 방법론을 실무에 적용**하여, 복잡한 비즈니스 로직 구현 및 아키텍처 설계 속도를 기존 대비 3배 이상 단축시킨 경험이 있습니다. AI의 출력을 검증하고 최적화하는 기술적 문해력을 갖추고 있습니다."

---

## 2. 기술 스택 (Technical Skills)
**전략:** AI 도구 활용 능력을 별도의 카테고리로 명시합니다.

*   **AI-Assisted Development:** Cursor, Gemini CLI, Claude 3.5 Sonnet, GitHub Copilot.
*   **Prompt Engineering:** 에이전트 기반 코드 생성 관리, Context 주입 및 프로젝트 가이드(GEMINI.md/CLAUDE.md) 설계.

---

## 3. 프로젝트 상세 (Project Experience)
**전략:** '어떻게(Methodology)'에 바이브 코딩을 녹여냅니다.

### [프로젝트 명: 사내 인트라넷 기반 파일 허브 및 공지사항 시스템]
*   **개발 방식:** AI Native 개발 (Vibe Coding 방법론 적용)
*   **주요 수행 내용:**
    *   **신속한 초기 아키텍처 설계:** AI 에이전트와의 협업을 통해 Spring Boot(Backend)와 React(Frontend) 간의 RESTful API 인터페이스 및 DB 스키마를 단시간 내에 설계 및 구축.
    *   **의도 기반 점진적 개발:** 자연어 요구사항 정의를 통해 복잡한 파일 업로드/다운로드 로직 및 JWT 인증 기반 보안 체계 구현.
    *   **품질 관리 및 디버깅:** AI가 생성한 코드의 구조적 결함을 파악하고, 테스트 시나리오를 작성하여 시스템 안정성 검증.
*   **성과:** 기획부터 배포(Docker)까지의 전체 개발 사이클을 1인 개발 체제에서 극단적으로 단축.

---

## 4. 기타 역량 (Strength)
**전략:** 바이브 코딩을 통해 얻은 소프트 스킬을 강조합니다.

*   **의사결정 능력:** AI가 제시하는 여러 구현 방식 중 프로젝트 상황(성능, 확장성)에 가장 적합한 방안을 선택하고 조율하는 아키텍트적 시각 보유.
*   **빠른 학습 및 적응력:** 새로운 기술 스택이나 복잡한 비즈니스 도메인도 AI를 활용해 빠르게 파악하고 실무 코드로 구현해내는 문제 해결 능력.

## 💡 면접 대비 팁
**Q: 바이브 코딩은 단순히 코드를 복사하는 것이 아닌가요?**
**A:** "바이브 코딩은 단순히 코드를 복사하는 것이 아니라, 개발자가 **'시스템 설계자'이자 '코드 리뷰어'**가 되어 AI에게 정확한 의도를 전달(Prompting)하고, 결과물의 품질을 기술적으로 검증(Verification)하는 고도의 협업 과정입니다. 이를 통해 단순 반복 코딩 시간을 줄이고, 비즈니스 로직의 핵심 가치와 사용자 경험(UX) 고도화에 더 집중할 수 있었습니다."
